glibc 升级

这周在对接个第三方的接口，需要用到JNI调用.so文件，想想调用第三方API应该挺简单，没想到这是个坑的开始。按照他们给的文档部署好，调用出现”GLIBC_2.7” not found 然后找找啊，找了各种文档，翻了各种墙，再装崩溃了一台服务器的前提下，终于找到了个方案
glibc是GNU发布的libc库，即c运行库。glibc是linux系统中最底层的api，几乎其它任何运行库都会依赖于glibc。

操作步骤

1、查看glibc版本  ldd --version查看
2、新建mytemp文件夹，mkdir home/mytemp
3、将glibc-2.7放到目录下解压 tar -zxvf glibc-2.7.tar.gz
4、进入到glibc-2.7目录  cd glibc-2.7
5、将glibc-linuxthreads-2.5.tar.bz2 放在glibc-2.7目录下，解压，tar -jxvf glibc-linuxthreads-2.5.tar.bz2
6、切换到mytemp目录下 cd ..
7、执行命令export CFLAGS="-g -O2 -march=i486"
8、执行命令glibc-2.7/configure --prefix=/usr --disable-profile --enable-add-ons --with-headers=/usr/include --with-binutils=/usr/bin --disable-sanity-checks
9、执行make
10、执行make install
11、看最后安装成功否，若最后没有出现错误，则重启机器  shutdow -r now
