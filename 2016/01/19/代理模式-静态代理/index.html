代理模式

抽象角色：通过接口或者抽象类声明业务方法
真实角色：继承或者实现抽象角色的方法
代理角色：实现抽象角色，通过真实角色的业务方法来实现抽象方法，并添加自己的操作
分为静态代理、动态代理
使用场景：一天你写了个接口，然后实现了具体的业务方法，很开心。第二天领导跟你说，要在调用这个方法之前增加的操作，然后你想着改源码，太low了，于是代理类就出来了。

静态代理

代码实现：需要改动的类较少，就这么来，简单高效，很hi

抽象角色:

public interface InterFaceRole{

    public void doSomething();

}

真实角色：

public class RealRole implements InterFaceRole{
    @Override
    public void doSomething() {
        System.out.println("真实角色处理任务了！！！");
    }
}

代理角色：

public class ProxyRole implements InterFaceRole{

    private RealRole realRole;

    public ProxyRole(RealRole realRole) {
        this.realRole = realRole;
    }
    @Override
    public void doSomething() {
        //添加你的方法
        System.out.println("代理类开始处理！！！！！！！！");
        realRole.doSomething();
    }

}
