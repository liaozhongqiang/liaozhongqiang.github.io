代理模式-动态代理-JDK

用了静态代理后，你发现是不是挺好，挺优雅。。。某日，领导让你修改1000个这样的方法，，，，然后你又去修改1000个这样的方法，这下要创建1000个类，，，那也是日了狗了。。。。。
这下动态代理出来了

JDK动态代理类

java.lang.reflect.Proxy

package java.lang.reflect;
public interface InvocationHandler {

    public Object invoke(Object proxy, Method method, Object[] args)
        throws Throwable;
}

抽象角色

package jdkproxy;

public interface InterFaceRole {

    public void doSomeThing();

}

具体角色

package jdkproxy;

public class RealRole implements InterFaceRole{

    @Override
    public void doSomeThing() {
        System.out.println("真实角色处理！！！！");
    }

}

代理角色

package jdkproxy;

import java.lang.reflect.InvocationHandler;
import java.lang.reflect.Method;

public class ProxyRole implements InvocationHandler{

    private Object RealRole;

    public ProxyRole(Object RealRole) {
        super();
        this.RealRole = RealRole;
    }

    @Override
    public Object invoke(Object proxy, Method method, Object[] args) throws Throwable {
        System.out.println("代理类操作！！！！！");
        Object obj=method.invoke(RealRole, args);
        return obj;
    }

}

调用

package jdkproxy;

import java.lang.reflect.Proxy;

public class Test {

    public static void main(String[] args) {
        InterFaceRole interFace=(InterFaceRole) Proxy.newProxyInstance(RealRole.class.getClassLoader(), RealRole.class.getInterfaces(), new ProxyRole(new RealRole()));
        interFace.doSomeThing();
    }

}

经过上述实验，代理类必须实现一个或几个接口，假如没有接口呢？改造一下调用方法如下：

package jdkproxy;

import java.lang.reflect.Proxy;

public class Test {

    public static void main(String[] args) {
        RealRole interFace=(RealRole) Proxy.newProxyInstance(RealRole.class.getClassLoader(),RealRole.class.getInterfaces(), new ProxyRole(new RealRole()));
        interFace.doSomeThing();
    }

}

执行发现会出错，

java.lang.IllegalArgumentException: jdkproxy.RealRole is not an interface

看来必须要有个接口。。能不能没有接口呢？请看下回分解cglib代理
